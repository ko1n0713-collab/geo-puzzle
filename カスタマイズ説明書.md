# 地理おちものパズル カスタマイズ説明書

このゲームに**問題(地形・分野)を足したり、遊びやすさを調整したりする方法**をまとめています。
プログラミングの知識は不要です。編集するのは `config.js` と `data.js` の2つだけです。

---

## 1. ファイルの一覧と役割

| ファイル | 役割 | 編集してよい? |
|---|---|---|
| `config.js` | 速さ・レベル・スコアなどの調整値 | ⭕ 自由に編集OK |
| `data.js` | 地理データ(都道府県・地形・地図記号) | ⭕ レビュー・修正OK |
| `index.html` | 画面とゲームの仕組み本体 | ❌ 編集しない(直したいときは依頼する) |
| `manifest.json` / `sw.js` / `icons/` | ホーム画面アプリ化のための設定 | ❌(ただし更新時に `sw.js` の `CACHE_VERSION` だけ上げる) |
| `データ確認表.md` | データの一覧表(自動生成) | ❌ 直接編集しない(下記 5. で作り直す) |
| `tools/generate-table.mjs` | 確認表を作り直すプログラム | ❌ |

## 2. よくある編集レシピ(data.js)

### 2-1. 対応する都道府県を直す

例:「◯◯川に△△県が抜けている」とき。`data.js` で地形を探し、`prefectures` に県の id を足します。

```javascript
{ id: "tone", name: "利根川", kana: "とねがわ", type: "river",
  prefectures: ["gunma", "saitama", "ibaraki", "chiba"],  // ← ここに足す・消す
  note: "流域面積日本一。坂東太郎と呼ばれる。" },
```

県の id は `data.js` の `prefectures:` の一覧にあります(例: 千葉県 = `chiba`)。

### 2-2. 地形(項目)を1つ足す

同じ種別のブロックをコピーして書きかえるのが簡単です。

```javascript
{ id: "kitakami2",            // ほかとかぶらない半角英数の名前
  name: "○○川",              // ブロックに表示される名前
  kana: "○○がわ",            // ふりがな
  type: "river",              // 種別(river / plain / basin / plateau / mountain)
  prefectures: ["iwate"],     // 対応する県の id(複数OK)
  note: "説明(消したとき豆知識として表示される)" },
```

**注意: 正確さが最優先です。** 地図帳で確認できない対応は入れないでください。

### 2-3. 新しい分野(種別)を足す — 例:「湖」

**手順は2つだけ**。プログラムの変更は不要で、色・凡例・消去・豆知識まで自動で反映されます。

1. `data.js` の `featureTypes:` に1行足す:

```javascript
{ id: "lake", label: "湖", color: "#26c6da", icon: "○", factTemplate: "{pref}にある" }
```

- `color` はブロックの色(インターネットで「カラーコード」と検索すると選べます)
- `factTemplate` は消したときの文。`{pref}` の部分が県名になります

2. `features:` にその種別の項目を足す:

```javascript
{ id: "biwako", name: "琵琶湖", kana: "びわこ", type: "lake",
  prefectures: ["shiga"], note: "日本一大きい湖。" },
```

地形以外の分野も同じ方法で足せます(考え方:「都道府県と結びつく知識」なら何でもOK)。

- 農産物: `factTemplate: "{pref}が生産量上位"` → りんご×青森・長野 など
- 伝統工芸品: `factTemplate: "{pref}の伝統工芸品"` → 西陣織×京都 など
- 世界遺産・旧国名なども同様

### 2-4. 地図記号を直す・足す

`data.js` の `mapSymbols:` にある `name` / `kana` / `note` は自由に直せます。
ここに載っている記号は「地図記号ずかん」と**レベル「地図記号」の出題**の両方に使われます
(記号の絵ブロックと名前ブロックをとなりに置くと消えるモード)。
`svg`(記号の絵)の編集はむずかしいので、記号の追加・修正は依頼してください。

## 3. レベルの編集(config.js の LEVELS)

レベルは上から順に Lv1, Lv2, … になります。1レベルはこの形です:

```javascript
{ name: "川と平野",                     // タイトル画面に出る名前
  pool: { types: ["river", "plain"] },  // 出題はんい(下の3つの書き方が使える)
  speedMs: 1500,                        // 1マス落ちる時間(小さいほど速い)
  hint: true },                         // ヒント機能(今後実装)
```

`pool`(出題はんい)の3つの書き方:

| 書き方 | 意味 | 例 |
|---|---|---|
| id の一覧 | 選んだ地形だけ | `pool: ["tone", "arakawa"]` |
| 種別指定 | その種別ぜんぶ(あとで足した項目も自動で入る) | `pool: { types: ["lake"] }` |
| `"all"` | 地形データぜんぶ | `pool: "all"` |
| `"mapSymbols"` | 地図記号モード(記号の絵×名前の一致で消える) | `pool: "mapSymbols"` |

県ブロックは出題はんいの地形から自動で決まるので、書く必要はありません。
レベルを増やしたいときは、ブロックごとコピーして `],` の前に足してください。

## 4. 遊びやすさの調整早見表(config.js)

| 変えたいこと | 直す場所 | 目安 |
|---|---|---|
| 落ちる速さ | 各レベルの `speedMs` | 大きいほどゆっくり(1500=ゆったり) |
| だんだん速くなるペース | `SPEEDUP_EVERY_PIECES` / `SPEEDUP_FACTOR` | FACTOR を 1 にすると速くならない |
| 消しやすさ(相手の出やすさ) | `PARTNER_BIAS` | 0.5 が標準。0.7 でかなり消しやすい |
| 県ブロックの量 | `PREF_RATIO` | 1.0 が標準(半々) |
| 豆知識の表示時間 | `FACT_DURATION_MS` | 2600 = 2.6秒 |
| 置き直しのゆとり | `LOCK_DELAY_MS` | 大きいほど接地後に動かせる |
| 盤面の大きさ | `COLS` / `ROWS` | 標準 6×12 |

## 5. 編集したあとにやること

1. **データを変えたら**: `データ確認表.md` を作り直す(パソコンに Node.js がある場合、リポジトリのフォルダで `node tools/generate-table.mjs`)。むずかしければ依頼でOK:
   > データ確認表を再生成してください
2. **公開中のゲームに反映するには**: `sw.js` の先頭の `CACHE_VERSION` を1つ上げて(v1→v2)、GitHub に push する。これも依頼でOK:
   > ◯◯を直したので、CACHE_VERSION を上げて push してください
3. iPad 側は、アプリを一度閉じて開き直すと新しい版になります(変わらないときはもう一度)。
